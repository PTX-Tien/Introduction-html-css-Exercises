Priority in CSS

In CSS, the priority of styles determines which styles are applied when multiple rules match the same element. The priority is determined by the specificity of the selectors and the order in which they are defined.

Specificity Levels

  1. Inline styles (highest priority)
  2. Internal styles
  3. External styles (lowest priority)

Inline styles have the highest priority because they are applied directly to the HTML element. Internal styles come next, as they are defined within the HTML document. External styles have the lowest priority since they are linked from an external file.

Example of Priority

Example 1: Inline vs Internal CSS

<- Internal CSS ->
<style>
    .example {
        color: blue;    /* Internal style */
    }
</style>

<- Inline CSS ->
<div class="example" style="color: red;">This text will be red.</div>

In this example, the text inside the <div> will be red because the inline style (style="color: red;") has a higher priority than the internal style (.example { color: blue; }).

Example 2: Internal vs External CSS

<- Internal CSS ->
<style>
    h4 {
        color: blue;    /* Internal style */
    }
</style>

<- External CSS ->
<link rel="stylesheet" href="styles.css">

<- styles.css ->
h4 {
    color: red;  /* External style */
}

<h4>This text will be green.</h4>

In this example, the text inside the <h4> will be red because if both styles had the same specificity, the one defined last would take precedence.

Order of Precedence

If multiple rules have the same specificity, the one that appears last in the CSS will take precedence. This means that if you define a style for an element multiple times, the last one will be applied.

Understanding Specificity

Specificity is calculated based on the types of selectors used. The more specific a selector is, the higher its priority. Here are some examples:

  • id selectors (e.g., #example) have higher specificity than class selectors (e.g., .example).
  • Class selectors have higher specificity than element selectors (e.g., div).
  • Inline styles have the highest specificity, overriding all other styles.

If two selectors have the same specificity, the one that appears last in the CSS will take precedence.

Practical Tips

  • Use inline styles sparingly, as they can make your code harder to maintain.
  • Prefer internal or external styles for better organization and reusability.
  • Be mindful of the order in which you define your styles to avoid unintended overrides.
  • Use browser developer tools to inspect elements and see which styles are being applied and overridden.

Conclusion

Understanding CSS priority is essential for effective styling of web pages. By knowing how specificity and order affect which styles are applied, you can create more predictable and maintainable CSS.
